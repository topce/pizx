# Self-Improvement Quality Scripts v2 — Design

**Date:** 2025-06-14  
**Status:** Approved  
**Context:** Extending the 6 baseline self-improvement scripts with quality/correctness-focused patterns

## Overview

Add 5 quality-focused scripts + 1 orchestrator runner to `self-improve-examples/`. These scripts generate `.patch` files (not auto-apply) so you review before applying with `git apply`.

## Architecture

```
self-improve-examples/
├── (6 existing baseline scripts)
├── review-debate.mjs               Δ Debate — 4-perspective code review
├── review-self-verify.mjs          Λ Pipeline — code-against-itself verification
├── hunt-bugs.mjs                   Φ Fleet — 6-category bug scan
├── heal-build.mjs                  Ρ Ralph Loop — broken build auto-healer
├── fuzz-adversarial.mjs            Δ Debate — attacker vs defender edge cases
├── quality-runner.mjs              Orchestrator — runs all quality scripts
└── patches/                        .patch output directory
    ├── review-debate/
    ├── review-self-verify/
    ├── hunt-bugs/
    ├── heal-build/
    └── fuzz-adversarial/
```

## `.patch` Format

All patches written to `patches/<script-name>/<timestamp>-<desc>.patch` with header:
- Generated by (script name)
- Timestamp
- Summary (one-line description)
- Confidence (0.0-1.0)
- Risk (LOW/MEDIUM/HIGH + rationale)
- Review note (specific thing to check before applying)

## Script Designs

### 1. review-debate.mjs
- **Pattern:** Δ Debate, 4 roles: security, performance, readability, correctness
- **Input:** `git diff` (staged + unstaged), optional `TARGET` env var
- **Flow:** Debate with 2 rounds → convergence synthesis → `.patch` per confirmed issue
- **Options:** `rounds: 2`, `mode: 'agent'`, `qualityCheck: true`

### 2. review-self-verify.mjs
- **Pattern:** Λ Pipeline, 2 stages
- **Stage 1:** Extract implied promises from code (type assertions, comments, docstrings)
- **Stage 2:** Verify each promise against actual code → flag broken promises → `.patch`
- **Difference from verify-spec-to-code.mjs:** checks code vs itself, not docs vs code

### 3. hunt-bugs.mjs
- **Pattern:** Φ Fleet, 6 parallel agents
- **Categories:** null/undefined, race conditions, resource leaks, swallowed errors, type safety, error propagation
- **Output:** Per-category `.patch` with file:line, severity, fix

### 4. heal-build.mjs
- **Pattern:** Ρ Ralph Loop
- **Input:** Failing test output (auto-captured or explicit)
- **Flow:** Analyze → Plan → Execute → Verify → Review loop, max 3 iterations
- **Options:** `maxIterations: 3`, `useTools: true`, optional `TEST_FILTER`
- **Output:** Cumulative `.patch` with Ralph audit trail

### 5. fuzz-adversarial.mjs
- **Pattern:** Δ Debate, 2 roles: attacker and defender
- **Flow:** Attacker generates edge cases → Defender produces fixes → 2 rounds → `.patch`
- **Output:** `.patch` containing both failing test AND fix
- **Options:** `rounds: 2`, optional `TARGET` env var

### 6. quality-runner.mjs
- **Pattern:** Orchestrator-style pipeline
- **Flow:** Runs debate → self-verify → hunt-bugs → (optional) heal-build → fuzz-adversarial
- Merges all `.patch` files, prints summary

## Scope Boundaries

**In scope:** 5 quality scripts + runner, `.patch` output, composable design  
**Out of scope:** CI/CD integration, auto-apply, Slack notifications, multi-project support, fuzzing with random inputs (semantic edge cases only)
